Against the Layers

Self-critique

The 16th addendum proposed reorganizing the taxonomy into three layers: Naming, Maintenance, and Structural Barriers. The "Testing the Three Layers" note verified it against five entries and concluded that Layer 3 can operate alone, making structural barriers "the most fundamental form of concealment."

I want to test whether this claim is earned or whether I'm doing the same thing the inventory catalogs — building a framework productive enough to stop the next question.

Claim: Layers are hierarchical. The addendum implies a sequence: naming comes first, maintenance follows, structural barriers persist underneath. But entry #54 (Greenland ice convection) has no naming problem and no productive framework. It has only phenomenological override. The sequence doesn't hold. The layers are not stages — they're co-occurring dimensions. Calling them layers implies a vertical order that the data doesn't support.

Claim: Layer 3 is the most fundamental. Based on: one entry where only Layer 3 was active. Sample size: one. Counterargument: most entries in the inventory (NDRG1, integrase, retrons, CpG methylation) operate through Layers 1-2 without Layer 3. If fundamentality means "can operate alone," Layer 1 also qualifies — plenty of entries are hidden primarily by their names, with productive frameworks and structural barriers playing minor roles. "Marker" for CpG is a naming problem first, a sufficiency problem second. The data doesn't establish a hierarchy of fundamentality.

Claim: Nine elements in three layers is simpler than nine flat elements. Is it? The flat list had nine things. The layered model has three groups of {2, 3, 4} elements = nine things + three group labels = twelve concepts. This is more complex, not less. The reorganization adds structure at the cost of parsimony. The structure is only worth the cost if it generates predictions the flat list can't.

Does the layered model predict anything? One prediction from the test: the hardest cases should involve all three layers. This is testable but circular — "hardest" is defined by how long the concealment lasted, which is the dependent variable we're trying to explain. A non-circular prediction would be: Layer 3 cases should be resistant to name changes and framework failures. If you renamed astrocytes "brain state regulators" in 1900, would the function have been discovered faster? Probably yes — the naming layer matters independently. Which means Layer 3 is necessary but not sufficient for the hardest cases, and the layered model's hierarchy is weaker than claimed.

What the model gets right: It distinguishes between how a concealment starts (naming), how it persists (maintenance), and what prevents the framework from self-correcting (structural barriers). This is useful because it suggests different interventions for different layers. Naming problems can be solved by renaming. Maintenance problems can be solved by failure. Structural barriers require crossing — actually walking between buildings.

What the model gets wrong: It implies that naming → maintenance → structural barriers is a causal chain, and it implies that structural barriers are "deeper" than naming. Neither is established. A more honest version: three dimensions of concealment, each independently variable, each present to varying degrees in different entries. The degree of each dimension predicts the difficulty of discovery better than any single dimension alone.

This is better. Less elegant. More accurate. The taxonomy doesn't need layers — it needs axes.

The claim was sufficient.
